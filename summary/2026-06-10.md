### Q1. JavaScript의 이벤트 루프(Event Loop) 동작 원리를 설명하시오

> **importance_score**: 4/5 | **빈출**: high | **출처**: [woowacourse-study/2024-woowa-sahon-interview-study](https://github.com/woowacourse-study/2024-woowa-sahon-interview-study)

**모범 답변:**
JavaScript는 싱글 스레드로 동작하지만, 이벤트 루프 덕분에 비동기 처리가 가능합니다. 콜 스택(Call Stack)에서 동기 코드가 모두 실행되면, 이벤트 루프는 먼저 **마이크로태스크 큐**(Promise, queueMicrotask)를 모두 비운 뒤, **태스크 큐**(setTimeout, setInterval 등)에서 하나씩 꺼내 실행합니다. 브라우저의 Web API가 비동기 작업(타이머, HTTP 요청 등)을 처리하고 완료된 콜백을 각 큐에 넣어주며, 이벤트 루프가 이를 콜 스택으로 옮겨 실행하는 구조입니다.

### Q2. Promise에 대해 설명하고 Callback function과의 차이점은 무엇인가요?

> **importance_score**: 4/5 | **빈출**: high | **출처**: [smee6/Frontend_Jr_Interview](https://github.com/smee6/Frontend_Jr_Interview)

**모범 답변:**
Promise는 비동기 작업의 최종 완료(fulfilled) 또는 실패(rejected)를 나타내는 객체로, `pending → fulfilled/rejected` 세 가지 상태를 가집니다. Callback은 중첩이 깊어질수록 "콜백 지옥"이 발생해 가독성과 에러 처리가 어렵습니다. Promise는 `.then().catch().finally()` 체이닝으로 이를 해결하고, `Promise.all()` / `Promise.race()` 등으로 병렬 처리도 쉽게 표현할 수 있습니다.
```js
// Callback Hell
getData(a => getMore(a, b => doSomething(b, c => {})));
// Promise 체이닝
getData().then(getMore).then(doSomething).catch(handleError);
```

### Q3. async/await 문법을 설명하고 Promise와 어떻게 다른가요?

> **importance_score**: 4/5 | **빈출**: high | **출처**: [smee6/Frontend_Jr_Interview](https://github.com/smee6/Frontend_Jr_Interview)

**모범 답변:**
`async/await`은 Promise를 기반으로 동작하는 문법적 설탕(syntactic sugar)입니다. `async` 함수는 항상 Promise를 반환하고, `await`은 Promise가 resolve될 때까지 해당 함수 내 실행을 일시 중단합니다. Promise 체이닝보다 동기 코드처럼 읽혀 가독성이 높고, `try/catch`로 에러 처리가 직관적입니다. 단, `await`은 `async` 함수 내에서만 사용 가능하며, 병렬 요청 시 `Promise.all()`과 조합하는 것이 효율적입니다.
```js
async function fetchUser() {
  try {
    const res = await fetch('/api/user');
    return await res.json();
  } catch (e) {
    console.error(e);
  }
}
```

### Q4. 마이크로태스크 큐(Microtask Queue)와 태스크 큐(Task Queue)의 차이점은 무엇인가요?

> **importance_score**: 4/5 | **빈출**: high | **출처**: [Esoolgnah/Frontend-Interview-Questions](https://github.com/Esoolgnah/Frontend-Interview-Questions)

**모범 답변:**
이벤트 루프는 콜 스택이 비워지면 **마이크로태스크 큐를 먼저, 전부** 처리한 뒤 태스크 큐에서 **하나씩** 꺼냅니다. 마이크로태스크 큐에는 `Promise.then/catch/finally`, `queueMicrotask`, `MutationObserver` 콜백이 들어가고, 태스크 큐(매크로태스크 큐)에는 `setTimeout`, `setInterval`, `I/O`, `UI 이벤트` 콜백이 들어갑니다. 따라서 동일 코드에서 `Promise.then` 콜백은 항상 `setTimeout` 콜백보다 먼저 실행됩니다.

### Q5. 실행 컨텍스트(Execution Context)란 무엇이며 콜 스택(Call Stack)과 어떤 관계가 있나요?

> **importance_score**: 4/5 | **빈출**: high | **출처**: [smee6/Frontend_Jr_Interview](https://github.com/smee6/Frontend_Jr_Interview)

**모범 답변:**
실행 컨텍스트는 코드가 실행되는 환경 정보(변수 환경, 스코프 체인, `this` 바인딩)를 담은 객체입니다. 전역 컨텍스트가 먼저 생성되고, 함수 호출마다 새 실행 컨텍스트가 **콜 스택에 push**됩니다. 함수 실행이 끝나면 **pop**되어 제거됩니다. 콜 스택이 가득 차면 스택 오버플로우가 발생하며, 비동기 콜백은 콜 스택이 비었을 때 이벤트 루프에 의해 스택에 올라옵니다.
