### Q1. CORS란 무엇이며, 어떻게 해결하나요?

> **importance_score**: 3/5 | **빈출**: high | **출처**: [smee6/Frontend_Jr_Interview](https://github.com/smee6/Frontend_Jr_Interview)

**모범 답변:**
CORS(Cross-Origin Resource Sharing)는 브라우저가 다른 출처(프로토콜·도메인·포트)의 리소스 요청을 차단하는 보안 정책입니다. 서버 측에서 `Access-Control-Allow-Origin` 응답 헤더를 설정해 허용 출처를 명시하면 해결됩니다. 개발 환경에서는 webpack devServer의 `proxy` 설정이나 NGINX 리버스 프록시로 우회할 수 있습니다. Preflight 요청(OPTIONS 메서드)은 실제 요청 전 브라우저가 서버 허용 여부를 미리 확인하는 과정입니다.

### Q2. HTTP와 HTTPS의 차이점은 무엇인가요?

> **importance_score**: 3/5 | **빈출**: high | **출처**: [smee6/Frontend_Jr_Interview](https://github.com/smee6/Frontend_Jr_Interview)

**모범 답변:**
HTTP는 데이터를 평문으로 전송하지만, HTTPS는 TLS(Transport Layer Security) 프로토콜로 암호화하여 전송합니다. HTTPS는 세 가지를 보장합니다: **기밀성**(도청 방지), **무결성**(데이터 변조 방지), **인증**(서버 신원 확인). 브라우저 주소창의 자물쇠 아이콘은 TLS 인증서가 유효함을 의미하며, HTTP/2는 HTTPS를 전제로 동작합니다.

### Q3. 쿠키(Cookie)와 세션(Session), 로컬 스토리지(LocalStorage)의 차이점은 무엇인가요?

> **importance_score**: 3/5 | **빈출**: high | **출처**: [smee6/Frontend_Jr_Interview](https://github.com/smee6/Frontend_Jr_Interview)

**모범 답변:**
| 구분 | 쿠키 | 세션 스토리지 | 로컬 스토리지 |
|------|------|--------------|--------------|
| 만료 | 설정한 만료일 | 탭 닫힘 시 삭제 | 명시적 삭제 전까지 유지 |
| 서버 전송 | 자동 전송(HTTP 헤더) | X | X |
| 저장 용량 | ~4KB | ~5MB | ~5MB |

쿠키는 인증 토큰 저장 시 `HttpOnly` + `Secure` 플래그로 XSS 공격을 방어할 수 있습니다. 로컬 스토리지는 서버에 전달되지 않아 민감한 정보 저장은 지양해야 합니다.

### Q4. XSS(Cross-Site Scripting)와 CSRF(Cross-Site Request Forgery)에 대해 설명하고, 방어 방법을 설명해주세요?

> **importance_score**: 3/5 | **빈출**: high | **출처**: [leegwae/getting-started](https://github.com/leegwae/getting-started)

**모범 답변:**
**XSS**는 공격자가 악성 스크립트를 웹 페이지에 삽입해 사용자 브라우저에서 실행시키는 공격입니다. 방어: 사용자 입력 이스케이프 처리, `Content-Security-Policy` 헤더 설정, `HttpOnly` 쿠키 사용.
**CSRF**는 인증된 사용자의 권한으로 의도치 않은 요청을 서버에 전송하는 공격입니다. 방어: CSRF 토큰 검증, `SameSite` 쿠키 속성 설정(`Strict` 또는 `Lax`), `Referer` 헤더 검증.

### Q5. 실시간 통신을 구현할 때 사용할 수 있는 기술(WebSocket, SSE, Long Polling)에 대해 설명하고 차이점을 비교해주세요?

> **importance_score**: 3/5 | **빈출**: high | **출처**: [leegwae/getting-started](https://github.com/leegwae/getting-started)

**모범 답변:**
- **Long Polling**: 클라이언트가 요청 후 서버가 응답을 늦게 주는 방식. HTTP 기반이지만 연결을 오래 유지해 지연이 발생합니다.
- **SSE(Server-Sent Events)**: 서버→클라이언트 단방향 스트림. HTTP/1.1 기반이며 자동 재연결을 지원합니다. 알림·피드 등에 적합합니다.
- **WebSocket**: 양방향 전이중(Full-duplex) 통신. 초기 HTTP 핸드셰이크 후 ws:// 프로토콜로 업그레이드됩니다. 채팅·게임·실시간 협업에 적합합니다.

```js
// SSE 사용 예
const es = new EventSource('/stream');
es.onmessage = (e) => console.log(e.data);
```
