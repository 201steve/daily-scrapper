### Q1. TypeScript와 JavaScript의 차이점은 무엇인가요?

> **importance_score**: 3/5 | **빈출**: high | **출처**: [spartacodingclub.kr](https://spartacodingclub.kr/blog/2024-frontend-jobinterview-question)

**모범 답변:**
TypeScript는 JavaScript의 슈퍼셋으로, 정적 타이핑을 지원합니다. JS는 런타임에야 타입 오류가 드러나지만, TS는 컴파일 단계에서 타입 오류를 미리 잡아냅니다. 또한 IDE 자동완성, 리팩터링 지원이 강화되어 대규모 팀 프로젝트에서 생산성과 안정성이 크게 향상됩니다. 단, 컴파일 단계가 추가되므로 빌드 시간이 늘어나는 트레이드오프가 있습니다.

### Q2. type alias와 interface의 차이점은 무엇인가요?

> **importance_score**: 3/5 | **빈출**: high | **출처**: [velog/@hjkwon](https://velog.io/@hjkwon/Typescript-Type-Alias-vs-Interface)

**모범 답변:**
`interface`는 선언 병합(Declaration Merging)이 가능하고 `extends`로 확장합니다. `type`은 중복 선언이 불가하며 `&`(인터섹션)으로 확장하고, 유니온·튜플·원시 타입 별칭 등 더 다양한 형태를 표현할 수 있습니다. 객체 형태 정의 시 라이브러리 확장이 필요하면 `interface`, 복잡한 타입 조합이 필요하면 `type`을 주로 선택합니다.
```ts
interface User { name: string }
interface User { age: number }  // OK: 선언 병합

type Point = { x: number } & { y: number }  // 인터섹션 타입
```

### Q3. TypeScript의 Generic이란 무엇이며, 어떻게 활용하나요?

> **importance_score**: 3/5 | **빈출**: high | **출처**: [github/woowacourse-study](https://github.com/woowacourse-study/2024-woowa-sahon-interview-study)

**모범 답변:**
Generic은 타입을 파라미터처럼 받아 재사용성을 높이는 기능입니다. 함수·클래스·인터페이스 선언 시 `<T>`로 타입 변수를 지정하고, 호출 시 구체적인 타입이 결정됩니다. `any`와 달리 타입 안전성을 유지하면서 다양한 타입에 동작하는 코드를 작성할 수 있습니다.
```ts
function identity<T>(value: T): T { return value }
identity<string>("hello")  // 반환 타입도 string으로 추론
```

### Q4. any, unknown, never, void 타입의 차이점을 설명해주세요.

> **importance_score**: 3/5 | **빈출**: high | **출처**: [velog/@jiseon-han](https://velog.io/@jiseon-han/%ED%83%80%EC%9E%85%EC%8A%A4%ED%81%AC%EB%A6%BD%ED%8A%B8-%EA%B0%84%EB%8B%A8-%EC%A0%95%EB%A6%AC-any-unknown-never)

**모범 답변:**
- **any**: 타입 검사를 완전히 비활성화. 어디에나 할당 가능하며 어떤 연산도 허용해 타입 안전성 포기.
- **unknown**: `any`처럼 모든 타입을 담을 수 있지만, 사용 전 타입 좁히기(type narrowing)가 강제됨. 안전한 대안.
- **never**: 절대 발생하지 않는 값의 타입. 무한루프 함수 반환값, 완전 exhaustive check 등에 사용.
- **void**: 의미 있는 값을 반환하지 않는 함수의 반환 타입. `undefined`를 반환하는 것과 유사하지만 의미상 "반환 없음"을 표현.

### Q5. 유틸리티 타입(Partial, Required, Pick, Omit, Exclude 등)을 사용해본 경험을 설명해주세요.

> **importance_score**: 3/5 | **빈출**: high | **출처**: [velog/@syoung125](https://velog.io/@syoung125/%EA%B0%9C%EB%85%90%EA%B3%B5%EB%B6%80-3.-Typescript%EA%B0%80-%EB%AD%98%EA%B9%8C-2-interface-type-alias-enum-Generic-Utility-TypesPick-Omit-Exclude)

**모범 답변:**
유틸리티 타입은 기존 타입을 변환해 새 타입을 만드는 내장 제네릭입니다. `Partial<T>`는 모든 프로퍼티를 선택적으로, `Pick<T, K>`는 특정 프로퍼티만 추출, `Omit<T, K>`는 특정 프로퍼티를 제외합니다. 실무에서는 API 응답 타입을 그대로 두고 `Partial`로 수정 폼 타입을 만들거나, `Pick`으로 컴포넌트 props 타입을 좁히는 패턴을 자주 사용합니다.
```ts
type UpdateUserDto = Partial<User>           // 모든 필드 선택적
type UserPreview = Pick<User, 'id' | 'name'> // 일부 필드만 사용
```
