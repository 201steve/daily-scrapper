### Q1. Core Web Vitals(LCP, INP, CLS)의 각 지표를 설명하고 각각의 개선 방법을 말씀해주세요.

> **importance_score**: 4/5 | **빈출**: high | **출처**: [youngju.dev](https://www.youngju.dev/blog/culture/2026-04-14-web-performance-optimization-core-web-vitals-guide-2025)

**모범 답변:**
- **LCP**(Largest Contentful Paint): 가장 큰 콘텐츠 요소가 렌더링되는 시간. 목표 2.5초 이하. → 이미지 preload, CDN, 서버 응답 시간 단축으로 개선
- **INP**(Interaction to Next Paint): 사용자 상호작용 후 다음 화면 업데이트까지 지연. 목표 200ms 이하. → 메인 스레드 블로킹 제거, 이벤트 핸들러 최적화
- **CLS**(Cumulative Layout Shift): 로드 중 예기치 않은 레이아웃 이동 누적. 목표 0.1 이하. → 이미지·광고에 `width`/`height` 명시, `font-display: optional` 사용

### Q2. 코드 스플리팅(Code Splitting)이란 무엇이고, React에서 어떻게 구현할 수 있나요?

> **importance_score**: 4/5 | **빈출**: high | **출처**: [velog/@bnb8419](https://velog.io/@bnb8419/%ED%94%84%EB%A1%A0%ED%8A%B8%EC%97%94%EB%93%9C-%EC%84%B1%EB%8A%A5-%EC%B5%9C%EC%A0%81%ED%99%94-Part.2)

**모범 답변:**
전체 번들을 한 번에 로드하는 대신, 필요한 시점에 필요한 코드만 내려받는 기법입니다. 초기 로딩 시간과 번들 크기를 크게 줄일 수 있습니다.
React에서는 `React.lazy()`와 `Suspense`를 조합해 컴포넌트 단위로 구현합니다:
```jsx
const LazyPage = React.lazy(() => import('./LazyPage'));
// <Suspense fallback={<Spinner />}><LazyPage /></Suspense>
```
라우트 단위 분할이 가장 효과적이며, Webpack의 Dynamic Import(`import()`)를 직접 활용할 수도 있습니다.

### Q3. React에서 불필요한 리렌더링을 방지하기 위한 방법들을 설명해주세요. (React.memo, useMemo, useCallback)

> **importance_score**: 4/5 | **빈출**: high | **출처**: [reactnext-central.xyz](https://reactnext-central.xyz/blog/react/memo-callback-hook)

**모범 답변:**
- **React.memo**: props가 변경되지 않으면 리렌더링을 건너뜀 (얕은 비교). 순수 컴포넌트에 적용
- **useMemo**: 계산 비용이 높은 값을 deps 변경 시에만 재계산. `const result = useMemo(() => heavyCalc(data), [data])`
- **useCallback**: 함수 참조를 deps 변경 시에만 새로 생성. 자식 컴포넌트에 함수 prop을 전달할 때 유용
> 주의: 실제 성능 병목이 확인된 경우에만 사용. 무분별한 적용은 불필요한 비교 비용을 추가함.

### Q4. 무한 스크롤 구현 시 발생할 수 있는 성능 문제와 이를 해결하는 방법은 무엇인가요?

> **importance_score**: 4/5 | **빈출**: high | **출처**: [nbcamp.spartaclub.kr](https://nbcamp.spartaclub.kr/blog/2025-%ED%94%84%EB%A1%A0%ED%8A%B8%EC%97%94%EB%93%9C-%EA%B0%9C%EB%B0%9C%EC%9E%90-%EB%A9%B4%EC%A0%91-%EB%AC%B8%EC%A0%9C%EC%9D%80%ED%96%89-%EB%A9%B4%EC%A0%91-%EA%BF%80%ED%8C%81%EA%B3%BC-%EC%98%88%EC%83%81-%EC%A7%88%EB%AC%B8-top-25-46939)

**모범 답변:**
무한 스크롤은 스크롤할수록 DOM 노드가 누적되어 메모리 증가와 렌더링 성능 저하를 유발합니다.
해결 방법:
1. **가상화(Virtualization)**: `react-window`, `@tanstack/virtual` 등으로 화면에 보이는 요소만 DOM에 유지
2. **IntersectionObserver**: `scroll` 이벤트 대신 사용해 스크롤 감지 비용 절감
3. **뷰포트 밖 요소 제거**: 화면 밖으로 벗어난 DOM 노드를 언마운트하고 동일 엘리먼트로 재사용

### Q5. Reflow(리플로우)와 Repaint(리페인트)의 차이점과 각각이 발생하는 시점을 설명해주세요.

> **importance_score**: 3/5 | **빈출**: high | **출처**: [Esoolgnah/Frontend-Interview-Questions](https://github.com/Esoolgnah/Frontend-Interview-Questions)

**모범 답변:**
- **Reflow**: DOM 요소의 크기·위치가 변경되어 레이아웃 전체를 다시 계산하는 과정. `width`, `height`, `padding`, `margin`, `position` 변경 시 발생. 비용이 매우 큼
- **Repaint**: 레이아웃 변경 없이 시각적 속성만 바뀌는 경우. `color`, `background-color`, `visibility` 변경 시 발생. Reflow보다 비용이 작음
최소화 방법: `transform`/`opacity` 속성 활용(GPU 레이어 분리), DOM 변경 일괄 처리, `DocumentFragment` 사용
