### Q1. useMemo와 useCallback의 차이점은 무엇이며, 각각 언제 사용하나요?

> **importance_score**: 5/5 | **빈출**: high | **출처**: [greatfrontend/top-reactjs-interview-questions](https://github.com/greatfrontend/top-reactjs-interview-questions)

**모범 답변:**
`useMemo`는 계산된 **값**을 메모이제이션하고, `useCallback`은 **함수** 자체를 메모이제이션합니다. `useMemo`는 대용량 데이터 필터링 같은 무거운 연산 결과를 캐시할 때 사용합니다. `useCallback`은 자식 컴포넌트에 함수를 props로 전달할 때 불필요한 리렌더링을 방지하기 위해 사용합니다. 두 훅 모두 의존성 배열의 값이 변경될 때만 재계산·재생성됩니다.
```js
const filtered = useMemo(() => data.filter(x => x.active), [data]);
const handleClick = useCallback(() => doSomething(id), [id]);
```

### Q2. useEffect와 useLayoutEffect의 차이점은 무엇인가요?

> **importance_score**: 5/5 | **빈출**: high | **출처**: [greatfrontend/top-reactjs-interview-questions](https://github.com/greatfrontend/top-reactjs-interview-questions)

**모범 답변:**
`useEffect`는 브라우저 페인트 **이후** 비동기적으로 실행되며, `useLayoutEffect`는 DOM 변경 직후 페인트 **이전**에 동기적으로 실행됩니다. 실무에서는 DOM 크기·위치 측정이나 스크롤 위치 복원처럼 화면 깜빡임을 막아야 할 경우 `useLayoutEffect`를 사용합니다. 그 외 대부분의 사이드이펙트(데이터 패칭, 이벤트 구독 등)는 `useEffect`로 충분합니다. SSR 환경에서는 `useLayoutEffect`가 경고를 발생시킬 수 있으므로 주의가 필요합니다.

### Q3. React에서 리렌더링(re-rendering)이 발생하는 조건은 무엇인가요?

> **importance_score**: 5/5 | **빈출**: high | **출처**: [greatfrontend/top-reactjs-interview-questions](https://github.com/greatfrontend/top-reactjs-interview-questions)

**모범 답변:**
리렌더링은 ① `setState` 호출로 **state가 변경**될 때, ② **부모 컴포넌트가 리렌더링**될 때(props 변경 여부와 무관), ③ 구독 중인 **Context 값이 변경**될 때 발생합니다. `React.memo`로 컴포넌트를 감싸면 props가 동일한 경우 리렌더링을 건너뛸 수 있고, `useMemo`·`useCallback`으로 props로 넘기는 값·함수를 안정화하면 불필요한 리렌더링을 줄일 수 있습니다.

### Q4. Context API 사용 시 주의해야 할 점은 무엇인가요?

> **importance_score**: 5/5 | **빈출**: high | **출처**: [greatfrontend/top-reactjs-interview-questions](https://github.com/greatfrontend/top-reactjs-interview-questions)

**모범 답변:**
Context 값이 변경되면 해당 Context를 구독하는 **모든 컴포넌트가 리렌더링**됩니다. 객체를 value로 그대로 전달하면 렌더링마다 새 참조가 생성되어 불필요한 리렌더링이 발생하므로, `useMemo`로 value를 안정화해야 합니다. 값이 자주 바뀌는 전역 상태에는 Zustand, Recoil 등 전용 상태관리 라이브러리 사용을 고려하세요. 또한 관심사가 다른 값은 Context를 분리해 영향 범위를 최소화하는 것이 좋습니다.

### Q5. Controlled 컴포넌트와 Uncontrolled 컴포넌트의 차이점은 무엇인가요?

> **importance_score**: 5/5 | **빈출**: high | **출처**: [greatfrontend/top-reactjs-interview-questions](https://github.com/greatfrontend/top-reactjs-interview-questions)

**모범 답변:**
**Controlled 컴포넌트**는 폼 값을 React state로 관리합니다. `value` prop과 `onChange` 핸들러를 함께 사용해 React가 항상 단일 진실 공급원(Single Source of Truth)이 됩니다. 실시간 유효성 검사, 조건부 버튼 활성화 등에 유리합니다. **Uncontrolled 컴포넌트**는 DOM이 값을 직접 관리하며 `useRef`로 필요할 때만 값을 읽습니다. 파일 업로드나 레거시 라이브러리 통합처럼 React의 제어가 어려운 경우에 사용합니다.
