### Q1. Flexbox와 CSS Grid의 차이점은 무엇이며, 각각 어떤 상황에 적합한가요?

> **importance_score**: 3/5 | **빈출**: high | **출처**: [velog - 프론트엔드 면접 질문 정리 Part2](https://velog.io/@developer-sora/%ED%94%84%EB%A1%A0%ED%8A%B8%EC%97%94%EB%93%9C-%EB%A9%B4%EC%A0%91-%EC%A7%88%EB%AC%B8-%EC%A0%95%EB%A6%AC-Part2HTMLCSS-Frontend)

**모범 답변:**
Flexbox는 **1차원(행 또는 열)** 레이아웃에 적합하고, Grid는 **2차원(행과 열 동시)** 레이아웃에 적합합니다.  
Flexbox는 내비게이션 바, 카드 목록처럼 단일 방향 배치에 쓰고, Grid는 전체 페이지 레이아웃이나 격자형 배치에 사용합니다.  
실무에서는 두 가지를 조합해 사용하는 경우가 많습니다. 예: Grid로 전체 레이아웃 잡고, 각 셀 내부는 Flexbox로 정렬.

### Q2. CSS 명시도(Specificity)란 무엇이며, 우선순위 계산 방식을 설명하세요.

> **importance_score**: 3/5 | **빈출**: high | **출처**: [velog - 프론트엔드 면접 질문 - CSS #2](https://velog.io/@chris/front-end-interview-handbook-css-2)

**모범 답변:**
명시도는 CSS 규칙이 충돌할 때 어떤 스타일을 적용할지 결정하는 점수 체계입니다.  
우선순위(높을수록 우선): `!important` > 인라인 스타일(1,0,0,0) > ID 선택자(0,1,0,0) > 클래스·속성·가상클래스(0,0,1,0) > 태그·가상요소(0,0,0,1).  
예: `#nav .link:hover` = (0,1,1,0), `div.link` = (0,0,1,1) → ID 선택자가 있는 쪽이 승리.  
`!important` 남용은 유지보수를 어렵게 하므로, 가능하면 명시도 구조를 올바르게 설계하는 것이 좋습니다.

### Q3. 시맨틱(Semantic) HTML이란 무엇이며, 왜 중요한가요?

> **importance_score**: 3/5 | **빈출**: high | **출처**: [velog - 프론트엔드 면접 질문 정리 Part2](https://velog.io/@developer-sora/%ED%94%84%EB%A1%A0%ED%8A%B8%EC%97%94%EB%93%9C-%EB%A9%B4%EC%A0%91-%EC%A7%88%EB%AC%B8-%EC%A0%95%EB%A6%AC-Part2HTMLCSS-Frontend)

**모범 답변:**
시맨틱 HTML은 콘텐츠의 **의미와 역할**을 명확히 전달하는 태그를 사용하는 것입니다 (`<header>`, `<nav>`, `<main>`, `<article>`, `<section>`, `<footer>` 등).  
중요한 이유 세 가지: ① **접근성** — 스크린 리더가 페이지 구조를 파악해 장애인 사용자에게 올바른 정보 전달, ② **SEO** — 검색 엔진이 콘텐츠 의미를 이해해 랭킹에 반영, ③ **유지보수성** — 코드 가독성 향상.  
`<div>` 남발보다 의미에 맞는 태그를 사용하는 것이 현대 웹 개발의 기본입니다.

### Q4. ARIA(Accessible Rich Internet Applications) 속성(role, aria-label, aria-labelledby, aria-hidden)의 역할을 설명하세요.

> **importance_score**: 3/5 | **빈출**: high | **출처**: [velog - 면접 준비 노트 (HTML, CSS)](https://velog.io/@dnr6054/fe-interview-html-css)

**모범 답변:**
ARIA는 시맨틱 HTML만으로 표현하기 어려운 UI 상태와 역할을 스크린 리더에 전달하는 속성 세트입니다.  
- `role`: 요소의 역할 명시 (예: `role="button"`, `role="dialog"`)  
- `aria-label`: 요소에 텍스트 레이블 직접 부여 (눈에 보이지 않는 경우)  
- `aria-labelledby`: 다른 요소의 ID를 참조해 레이블로 사용  
- `aria-hidden="true"`: 스크린 리더에서 해당 요소를 무시하게 함 (장식용 아이콘 등)  
원칙: **"Native first, ARIA second"** — 시맨틱 태그로 해결 가능하면 ARIA보다 시맨틱 태그를 우선 사용.

### Q5. CSS position 속성의 종류(static, relative, absolute, fixed, sticky)와 각각의 차이를 설명하세요.

> **importance_score**: 3/5 | **빈출**: high | **출처**: [velog - 프론트엔드 면접 질문 정리 Part2](https://velog.io/@developer-sora/%ED%94%84%EB%A1%A0%ED%8A%B8%EC%97%94%EB%93%9C-%EB%A9%B4%EC%A0%91-%EC%A7%88%EB%AC%B8-%EC%A0%95%EB%A6%AC-Part2HTMLCSS-Frontend)

**모범 답변:**
- `static`: 기본값. 문서 흐름에 따라 배치, top/left 등 속성 무시.  
- `relative`: 원래 위치 기준으로 오프셋 이동. 공간은 유지됨.  
- `absolute`: 가장 가까운 `position: non-static` 조상 기준으로 배치. 문서 흐름에서 제거됨.  
- `fixed`: 뷰포트 기준으로 고정. 스크롤해도 위치 유지.  
- `sticky`: 스크롤 시 임계점 도달 전엔 `relative`, 도달 후엔 `fixed`처럼 동작. 스크롤 헤더에 자주 사용.
