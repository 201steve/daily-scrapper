## 작업 순서

1. `raw/` 폴더에서 가장 최근에 추가된 `YYYY-MM-DD.md` 파일을 찾아 읽어라.
2. 같은 스킬 폴더의 `template.md` 파일을 읽어 출력 형식을 파악하라.
3. `data/sent-questions-log.md` 파일을 읽어라.
   - 파일이 없으면 빈 로그로 간주하고 계속 진행하라.
4. raw 파일의 질문 목록에서 sent-questions-log에 이미 기록된 질문을 제외하라.
   - 질문 텍스트가 동일하거나 의미상 거의 동일하면 중복으로 간주하라.
5. 남은 질문 중 아래 기준으로 **importance_score**를 산출하라 (가산 방식, 최대 5점):
   - 빈출(`frequency: high`): +2점
   - React 관련 카테고리 (React/컴포넌트): +1점 (취업 시장 수요 반영)
   - JavaScript/비동기 카테고리: +1점 (핵심 기초)
   - 실무 연관성 높은 개념 (렌더링 최적화, 상태관리, 네트워크 흐름 등): +1점
   - 최근 2025년 이후 출처 확인된 질문: +1점
6. importance_score 내림차순으로 정렬하고, 동점 시 빈출(`high`) 우선으로 상위 **5개**만 선별하라.
7. 선별된 5개 질문에 대해 각각 **모범 답변 초안**을 작성하라:
   - 핵심 개념 위주로 간결하게 (3-5줄 이내)
   - 실무 맥락이나 예시 코드가 있으면 포함
8. `template.md` 형식에 맞춰 한국어로 작성하라.
9. 결과를 `summary/YYYY-MM-DD.md` 경로로 새 파일을 생성하라.
   - 날짜는 raw 파일과 동일하게 사용
10. `data/sent-questions-log.md`에 오늘 발송한 5개 질문을 추가하라:
    - 형식: `- YYYY-MM-DD | (카테고리) | (질문 내용)`
    - 파일이 없으면 새로 생성하라.
11. 변경된 파일(`summary/YYYY-MM-DD.md`, `data/sent-questions-log.md`)을 커밋·푸시하라.
    - 커밋 메시지: `brief: YYYY-MM-DD`

## 요약 원칙

- raw에 없는 질문이나 답변을 만들어내지 말 것
- 출처 링크는 raw 파일에 있는 URL을 그대로 사용
- 모범 답변은 지나치게 길지 않게 (슬랙에서 읽기 좋은 분량)

## 금지 사항

- 템플릿 형식 임의 변경 금지
- sent-questions-log 업데이트 누락 금지
- 이미 발송된 질문 재발송 금지
- importance_score 5점 초과 금지
