## 주제

**프론트엔드 개발자 면접 질문 큐레이션** — 실제 면접에 출제된 질문 중심으로 카테고리별 로테이션 수집

## 카테고리 로테이션

오늘 날짜(KST)의 요일에 따라 수집할 카테고리를 결정하라:

| 요일 | 카테고리 |
|------|----------|
| 월   | 브라우저 / 렌더링 |
| 화   | React / 컴포넌트 |
| 수   | JavaScript / 비동기 |
| 목   | 성능 최적화 |
| 금   | 네트워크 / HTTP |
| 토   | CSS / HTML / 접근성 |
| 일   | TypeScript / 기타 |

## 수집 방법

1. 한국 시간 기준 오늘 날짜와 요일을 확인하여 수집 카테고리를 결정하라.
2. WebSearch로 아래 키워드 패턴으로 검색하라:
   - `프론트엔드 면접 질문 [카테고리명] 2024 OR 2025`
   - `[카테고리명] 면접 후기 질문`
   - `프론트엔드 기술 면접 [카테고리명]`
3. 검색 결과 중 다음 출처를 우선하라:
   - 벨로그(velog.io), 티스토리, 네이버 블로그, GitHub
   - 실제 면접 후기가 담긴 글 (면접 경험 공유, 질문 목록 정리 포스트)
4. WebFetch로 상위 결과 3-5개 페이지를 열어 질문 항목을 추출하라.
5. 각 질문에 대해 아래 정보를 수집하라:
   - 질문 내용 (`question`)
   - 카테고리 (`category`)
   - 출처 URL (`source_url`)
   - 빈출 여부 (`frequency`): 여러 출처에서 반복 등장하면 `high`, 한 곳에서만 보이면 `low`
   - 수집 날짜 (`collected_date`, ISO 8601)
6. 수집된 질문 중 중복(동일하거나 거의 동일한 질문)을 제거하라.
7. 최소 10개, 최대 20개 질문을 수집하라.
8. 결과를 `raw/YYYY-MM-DD.md` 경로로 새 파일을 생성하고 커밋·푸시하라.
   - YYYY-MM-DD는 오늘의 한국 날짜
   - 커밋 메시지: `scrap: YYYY-MM-DD`

## 출력 파일 형식

```
# Raw — YYYY-MM-DD (카테고리명)

## 수집 질문

### 1. (질문 내용)
- 카테고리: (category)
- 출처: (source_url)
- 빈출: (high / low)
- 수집일: (collected_date)

### 2. ...
```

## 금지 사항

- 질문에 대한 답변 포함 금지 (답변 작성은 briefing 단계에서 수행)
- 출처 불명확한 질문 수집 금지
- 카테고리와 무관한 질문 수집 금지
- 중복 질문 중복 수집 금지
